# KSQ 交互流程设计方案 (v2)

> KSQ = Key Screening Questions，AI 从简历中提取的 1-3 道核心验证题。
> 采用**两层结构**：重点考察（白盒可改）+ 基础覆盖（自动完成）。

---

## 一、核心流程

```
上传简历 → AI 生成 KSQ 确认卡片 → 招聘者确认/修改 → AI 开始面试 → 面试后生成报告
```

整个流程在 **EileenSidebar** 对话流中完成，不跳转新页面。

## 三、KSQ 题目推荐逻辑

### 重点考察题（1-3 道，来自简历分析）

| 信号来源 | 推荐规则 | 示例 |
|---------|---------|------|
| 简历异常 | 空窗期、频繁跳槽、时间重叠 → "动机核实"题 | "离职动机核实" |
| 量化声称 | 声称大规模、高 ROI → "实操验证"题 | "微前端接入规模验证" |
| 核心技能 | 从职位名推断关键技能 → "深度考察"题 | "React 项目经验深度" |

MVP 不依赖 JD 输入，纯从简历提取。

### 基础覆盖项（固定 4 项，自动完成）

| 覆盖项 | 对应 kaocha.md | AI 考察方式 |
|--------|---------------|-----------|
| 求职意向确认 | 一.2 求职核心匹配 + 一.3 地域适配 | 直接询问薪资/到岗/地域 |
| 工作经历核实 | 二.2 经验与成果匹配 | 针对简历各段经历追问细节 |
| 沟通表达观察 | 四.1 沟通表达能力 | 全程行为信号采集（语速/逻辑/停顿） |
| 职业动机与稳定性 | 五.2 职业动机与稳定性 | 询问求职动机、职业规划 |

## 四、面试后报告呈现（同样两层）

候选人速览卡结构（基础覆盖整合进 AI 判断区，消除与旧 L1 标签的重复）：

```
┌ AI 判断：🟡 建议面试，但需重点关注 ─────┐
│  "技术扎实但离职动机含糊..."              │
│                                          │
│  📑 基础覆盖                              │
│  ✓ 薪资匹配 · ✓ 学历达标 · ✓ 到岗可接受   │
│  ✓ 沟通表达正常 · ✓ 工作经历连贯           │
│  ⚠ 地域偏好未确认                         │
├──────────────────────────────────────────┤
│  🔍 重点考察回答                          │
│  1️⃣ React 项目经验  ✅ 达标              │
│     → 能说出 FCP 提升 40%，时间切片原理    │
│  2️⃣ 微前端架构实操  ⚠️ 存疑              │
│     → 说"十几二十个"但给不了具体数          │
│  3️⃣ 离职动机核实   ⚠️ 存疑               │
│     → 语速变慢，未正面回答裁员比例          │
├──────────────────────────────────────────┤
│  💬 面试时建议追问  2 项                   │
│  ...                                     │
└──────────────────────────────────────────┘

底部 sticky bar：[淘汰] [待定] [通过]
```

要点：
- 旧的 `✓ 薪资匹配 · ✓ 学历达标 · ⚠ 到岗时间需确认` 标签行被整合进「基础覆盖」
- 基础覆盖紧跟 AI 总结文案，形成完整的 AI 判断区
- "没问题" 也展示出来——告诉招聘者 "AI 检查了，都没问题"
