# 🧩 IHR Chrome 插件：外围与基础建设规划 (Peripheral Features Plan)

作为一款面向专业 HR 招聘场景的生产力工具，Chrome 插件除了核心的“简历解析”和“交互底座（Eileen Sidebar）”外，还需要完善的外围生态建设才能达到**不仅能用，还能留存、易传播**的商业级产品标准。

基于您的初步梳理，结合资深浏览器插件开发经验，我将从**用户生命周期**的角度，将插件外围建设分为五大模块：

---

## 一、 认知与获取（获取阶段）

1. **Chrome Web Store 详情页（上架材料）**
   - **核心物料**：Logo（多尺寸：16/32/48/128）、促销大图（Promo Tile 440x280）、高清截图（说明核心使用场景）、介绍视频（可选，但极大提升转化率）。
   - **文案**：核心价值主张（Value Proposition）、功能列表、支持的招聘平台（Boss、猎聘等）。
   - **合规要求**：必须提供**隐私政策（Privacy Policy）页面链接**（Chrome 商店审查极其严格），必须声明插件请求各项权限（如 `activeTab`, `storage`）的正当理由。

2. **独立产品落地页 (Landing Page)**
   - **域名与承载**：例如 `nexus.ihr.ai/extension`。
   - **内容**：产品功能介绍（动图演示）、**一键安装按钮**（跳转 Chrome 商店）、用户案例、版权信息、联系方式。
   - **SEO 优化**：针对“HR 招聘提效插件”、“简历解析插件”等关键词优化。

---

## 二、 激活与 onboarding（初次使用）

1. **安装成功后的欢迎页 (Welcome / Success Page)**
   - **触发机制**：监听 `chrome.runtime.onInstalled` 事件，当 `reason === 'install'` 时自动在新标签页打开。
   - **内容**：恭喜安装、功能总览。
   - **核心动作**：引导用户进行**登录/注册**（结合下文的账号体系）。

2. **账号注册与通行证体系 (Auth System)**
   - **方案设计**：由于我们有 Web 端（现在的 Next.js 项目），插件可以与 Web 端共享账号体系。
   - **登录态同步 (SSO)**：用户在 Web 端登录后，插件可以通过传递 Token（如基于 Cookie 或经由网页与插件通信 `externally_connectable`）自动登录；或者提供一个独立的统一登录弹窗。
   - **会话管理**：Token 存在插件的 `chrome.storage.local` 中，保障安全性。

3. **新手引导 (Interactive Onboarding / Tour)**
   - 在用户第一次打开招聘网站（如 Boss 直聘）时，通过气泡/高亮（Spotlight/Tooltip）引导用户：“点击这里唤起 Eileen”、“这里是简历解析按钮”。
   - 避免长篇大论，采用**渐进式引导（Progressive Disclosure）**。

---

## 三、 使用与留存（日常使用）

1. **状态管理与侧边栏常驻 (Sidebar Management)**
   - **侧边栏开关状态记忆**：用户关闭了侧边栏，刷新页面后是否保持关闭？通过 `chrome.storage.sync` 记录用户的偏好设置（如展开/折叠、暗黑模式偏好等）。

2. **版本更新与通知 (Update & Migration)**
   - 当插件自动升级时（监听 `reason === 'update'`），如果是大版本更新，可以推送一条系统通知（Chrome Notification API），或者在侧边栏顶部显示一个微小的“更新公告（What's New）”横幅。

3. **异常监控与遥测 (Telemetry & Error Tracking)**
   - 插件运行在用户的各种复杂环境中（不同分辨率、不同浏览器版本、网络波动），**必须接入报错拦截体系**（如 Sentry）。否则插件崩了你也不知道。
   - 基础的使用量遥测：例如记录“用户点击了多少次解析简历”，用于后续迭代。

---

## 四、 传播与防流失（生命周期末端）

1. **求助与反馈入口 (Feedback Loop)**
   - 在侧边栏的隐蔽角落（如设置菜单下）放一个“意见反馈/报告 Bug”的入口。
   - 如果用户遇到阻碍没地方反馈，他们会去 Chrome 商店打 1 星。

2. **卸载问卷 (Uninstall Survey)**
   - 极核动作：使用 `chrome.runtime.setUninstallURL("https://your-domain.com/uninstall-survey")`。
   - 当用户右键卸载插件时，会自动打开这个网页。用简单的单选题收集流失原因（“太卡了”、“不需要了”、“找不到某功能”、“有Bug”），这对产品改进价值巨大。

---

## 五、 实施构建计划建议

为了不阻碍主核心流程，我建议将这些“外围建设”分步骤落实：

### 阶段 1：MVP 必备外围（首个可用版本必须有）
* [ ] 账号登录打通（保证数据归属权，插件能在 Web 拿到鉴权）。
* [ ] 安装后的引导页（解释怎么用，且作为强制登录的卡口）。
* [ ] 隐私政策文档（上架必备，没有任何商量余地）。

### 阶段 2：体验与留存基建（核心流程跑通后补全）
* [ ] 新手交互式引导（Driver.js 等类似的 Tooltip 引导）。
* [ ] 错误监控接入（Sentry）。
* [ ] 卸载跳转问卷页面（挂一个简单的问卷星或 Typeform 链接即可）。

### 阶段 3：商业化与传播
* [ ] 独立的产品 Landing Page 构建。
* [ ] Chrome Web Store 专业级素材制作。
* [ ] 更新日志触达。
